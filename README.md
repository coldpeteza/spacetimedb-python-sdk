# SpacetimeDB SDK for Python

## Overview

This repository contains the [Python](https://python.org/) SDK for SpacetimeDB. The SDK allows to interact with the database server and is prepared to work with code generated from a SpacetimeDB backend code.

## Documentation

The Python SDK has a [Quick Start](https://spacetimedb.com/docs/client-languages/python/python-sdk-quickstart-guide) guide and a [Reference](https://spacetimedb.com/docs/client-languages/python/python-sdk-reference).

## Installation

The SDK is available as a [PyPi package](https://pypi.org/project/spacetimedb-sdk/). To install it, run the following command:

```bash
pip install spacetimedb-sdk
```

Runtime dependencies (installed automatically by pip):

- `websocket-client` ‚Äî used by the low-level synchronous `SpacetimeDBClient`
- `websockets >= 10.0` ‚Äî used by `SpacetimeDBAsyncClient` for native async I/O

## Authentication

SpacetimeDB identifies clients by an **auth token**:

- **First connection** ‚Äî pass `auth_token=None`. The server assigns a new identity and returns a fresh token via the `on_connect` callback. Save this token (e.g. with `local_config.set_token(auth_token)`) and reuse it on subsequent runs so your identity persists across sessions.
- **SpacetimeDB Cloud (OIDC/JWT)** ‚Äî tokens that look like `eyJ‚Ä¶` are automatically sent as `Authorization: Bearer <token>`.
- **Self-hosted / legacy tokens** ‚Äî all other tokens are sent as `Authorization: Basic base64("token:<token>")`.

## Usage

The Python SpacetimeDB SDK uses the `asyncio` package to provide an event-driven interface backed by an async-native WebSocket connection.

### Connecting to SpacetimeDB

Create a `SpacetimeDBAsyncClient` instance, passing the `module_bindings` package generated by the SpacetimeDB CLI:

```python
from spacetimedb_sdk.spacetimedb_async_client import SpacetimeDBAsyncClient

spacetime_client = SpacetimeDBAsyncClient(module_bindings)
```

Call `run()` to connect and start processing events. The coroutine does not return until the client is closed.

Parameters:

- `auth_token` ‚Äî authentication token (see [Authentication](#authentication) above; use `None` for a new identity).
- `host` ‚Äî `host:port` of the SpacetimeDB server, **without** a scheme prefix (e.g. `"localhost:3000"`).
- `address_or_name` ‚Äî module name or address to connect to.
- `ssl_enabled` ‚Äî `True` to use `wss://` (required for SpacetimeDB Cloud), `False` for `ws://`.
- `on_connect` ‚Äî callback invoked with `(auth_token, identity)` once the connection is established.
- `subscription_queries` ‚Äî list of SQL queries to subscribe to on connect.

Example:

```python
import asyncio
import spacetimedb_sdk.local_config as local_config

def on_connect(auth_token, identity):
    local_config.set_token(auth_token)   # persist token for next run
    print(f"Connected as {identity}")

asyncio.run(
    spacetime_client.run(
        local_config.get_token(),        # auth_token; None on first run
        "localhost:3000",                # host:port, no scheme
        "chatqs",                        # module name or address
        False,                           # ssl_enabled (True for Cloud)
        on_connect,
        ["SELECT * FROM User", "SELECT * FROM Message"],
    )
)
```

### Listening to events

Register callbacks on the `SpacetimeDBAsyncClient` instance or on auto-generated table classes:

- `register_on_subscription_applied(callback)` ‚Äî called once when the initial data snapshot arrives after subscribing.
- `schedule_event(delay_secs, callback, *args)` ‚Äî fire a callback after `delay_secs` seconds. Call `schedule_event` again from inside the callback to repeat it.

Register for row-level changes on a table via the auto-generated class:

- `register_row_update(callback)` ‚Äî invoked for every insert, update, or delete. The callback receives `(row_op, row_old, row, reducer_event)`:
  - `row_op` ‚Äî `"insert"`, `"update"`, or `"delete"`
  - `row_old` ‚Äî previous row value (`None` for inserts)
  - `row` ‚Äî new row value (`None` for deletes)
  - `reducer_event` ‚Äî the `ReducerEvent` that caused the change, or `None` for subscription-driven updates

Example:

```python
def on_message_row_update(row_op, message_old, message, reducer_event):
    if reducer_event is not None and row_op == "insert":
        print_message(message)

Message.register_row_update(on_message_row_update)
```

> **Event tables** ‚Äî some generated classes carry `is_event_table = True`. Their rows are ephemeral: `register_row_update` callbacks fire as normal, but the rows are **not** stored in the local cache and will not appear via `iter()` or `filter_by_*()`.

### Accessing the client cache

When you run the CLI `generate` command, SpacetimeDB creates a class for each table. Use these to query the local cache:

- `filter_by_COLUMN(value)` ‚Äî returns rows matching `column == value`.
- `iter()` ‚Äî returns an iterator over all cached rows.

Example:

```python
from module_bindings.user import User

my_user = User.filter_by_identity(local_identity)

for user in User.iter():
    print(user.name)
```

### Calling Reducers

Call the auto-generated function in the reducer module. Pass an optional `then` callback to be notified when the server processes the call:

```python
import module_bindings.send_message_reducer as send_message_reducer

# Fire-and-forget
send_message_reducer.send_message("Hello World!")

# With result callback (then= is optional)
send_message_reducer.send_message(
    "Hello World!",
    then=lambda evt: print(f"Failed: {evt.message}") if evt.status == "failed" else None,
)
```

The `then` callback receives a `ReducerEvent` with:

- `status` ‚Äî `"committed"` or `"failed"`
- `message` ‚Äî server error message (non-empty when `status == "failed"`)
- `reducer_name` ‚Äî name of the reducer that was called
